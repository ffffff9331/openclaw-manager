# openclaw manager 优化计划

## 目标
解决 Windows 端问题，精简架构，提升性能

---

## 阶段 1：紧急修复（1-2天）

### 1.1 Windows 端命令调用修复
- [ ] 审计所有 Tauri Command 调用
- [ ] 替换 macOS 特有的 shell 调用为跨平台方案
- [ ] 修复频繁开终端窗口问题
- [ ] 添加异步处理，避免主线程阻塞

### 1.2 无响应问题排查
- [ ] 检查所有同步调用点
- [ ] 优化长时间运行的操作
- [ ] 添加超时机制
- [ ] 改进错误处理

**验收标准：**
- Windows 端可正常连接 gateway
- 不再频繁弹出终端窗口
- app 响应流畅，无卡顿

---

## 阶段 2：代码精简（3-5天）

### 2.1 移除冗余代码
- [ ] 删除未使用的 hooks
- [ ] 合并重复的实例检测逻辑（新旧接口统一）
- [ ] 清理废弃的组件和服务
- [ ] 移除调试代码和注释

### 2.2 简化实例管理
- [ ] 统一实例检测流程
- [ ] 简化状态管理
- [ ] 减少不必要的 API 调用
- [ ] 优化持久化逻辑

### 2.3 依赖优化
- [ ] 审计 package.json，移除未使用的包
- [ ] 检查是否有更轻量的替代方案
- [ ] 优化 bundle 大小

**验收标准：**
- 代码量减少 30%+
- node_modules 减少 20%+
- 构建产物减少 15%+

---

## 阶段 3：架构优化（5-7天）

### 3.1 组件合并
- [ ] 合并功能相似的页面
- [ ] 减少组件层级
- [ ] 统一样式系统

### 3.2 状态管理简化
- [ ] 扁平化 store 结构
- [ ] 减少不必要的状态
- [ ] 优化更新频率

### 3.3 性能优化
- [ ] 启用 React.memo
- [ ] 优化渲染逻辑
- [ ] 添加虚拟滚动（如需要）
- [ ] 优化构建配置

**验收标准：**
- 启动时间减少 40%+
- 内存占用减少 30%+
- 操作响应时间 < 100ms

### 4.1 跨平台测试
