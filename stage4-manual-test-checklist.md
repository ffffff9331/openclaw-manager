## 验收用例

### A. 本机实例检测入口

#### A1 打开添加实例弹窗
- 入口可见
- 点击后弹出 `AddInstanceModal`
- 无报错、无空白

预期：
- 可正常打开弹窗
- 默认字段完整
- 默认类型与默认地址提示合理

#### A2 检测本机实例
- 在添加实例弹窗内触发“检测本机实例”动作

预期：
- 若本机 OpenClaw 可访问，应出现可加入结果
- 若不可访问，应给出明确失败/无结果提示
- 不应卡死、不应静默失败

#### A3 一键加入本机实例
- 对检测出的本机实例执行加入

预期：
- 新实例被保存
- 实例列表可见
- 类型 / 地址 / 来源口径正确
- 状态至少为 `online` 或 `unknown`，不能出现明显错乱

---

### B. 手动添加实例增强

#### B1 类型切换
依次切换：
- `local`
- `docker`
- `nas`
- `remote`

预期：
- 地址 placeholder 会随类型变化
- 辅助说明文案与类型匹配
- 不出现字段残留错位

#### B2 手动添加 local
输入本机地址并保存。

预期：
- 保存成功
- 列表中实例类型显示为 `local`
- 当前实例可切换到该项

#### B3 手动添加 docker
输入 Docker 场景地址并保存。

预期：
- 保存成功
- 列表中实例类型显示为 `docker`
- 引导文案仍与 Docker 场景匹配

#### B4 手动添加 nas
输入 NAS 场景地址并保存。

预期：
- 保存成功
- 列表中实例类型显示为 `nas`

#### B5 手动添加 remote
输入远端地址并保存。

预期：
- 保存成功
- 列表中实例类型显示为 `remote`

#### B6 非法输入拦截
尝试：
- 空名称
- 空地址
- 明显非法地址

预期：
- 有清晰校验提示
- 不应保存出脏实例

#### C1 实例列表展示

### D. 局域网发现入口

#### D1 mDNS 开关
在设置页查看并切换局域网发现（mDNS）开关。

预期：
- 开关可操作
- 风险提示文案存在
- 默认口径保守，不伪造发现结果

#### D2 未开启时点击扫描
预期：
- 给出“请先开启局域网发现”之类提示
- 不直接报错

#### D3 开启后执行扫描
预期：
- 扫描动作能执行
- 扫描中 / 扫描后状态文案合理
- 若无结果：明确提示未发现
- 若有结果：展示结果卡片

#### D4 发现结果保存为实例
对任一候选结果执行保存。

预期：
- 可保存为实例
- 来源标记为 `discovered`
- 保存后可在实例列表看到

---

### E. 在线状态刷新机制

#### E1 初始状态展示
预期：
- 实例状态显示为 `online / offline / unknown`
- 文案与颜色一致

#### E2 手动刷新
使用侧栏或相关入口刷新实例状态。

预期：
- 状态会重新探测
- 在线实例应尽量更新为 `online`
- 不在线实例应更新为 `offline` 或在 web preview 下合理保留 `unknown`

#### E3 轻量轮询
保持页面停留一段时间，观察状态是否自动刷新。

预期：
- 不需要用户手点也会定期刷新
- 不应出现高频抖动
- 不应引发明显卡顿

#### E4 Overview 汇总口径
查看 `OverviewPage` 中在线/离线/未知汇总。

预期：
- 汇总数量与实例列表一致
- 不出现明显统计错位
