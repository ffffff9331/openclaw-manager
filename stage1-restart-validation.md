## 三、人工实测前置条件

在关闭应用前，先准备一个可识别的测试状态：

### 1. 实例准备
至少保证存在：
- 本机实例
- 1 个手动新增的远端 / Docker / NAS 实例

建议测试实例示例：
- 名称：`阶段一恢复测试实例`
- 地址：`http://192.168.1.20:18789/`（或实际内网测试地址）

### 2. 当前实例切换
将当前实例切换到：
- 非默认本机实例

### 3. 设置项改动
至少改动一项**当前版本真实生效**的设置，例如：
- 深色模式开关
- 局域网访问开关（`allowLanAccess`）
- 白名单开关
- 文件访问开关

说明：
- `mDNS` 当前仍属于“预留偏好 / 规划中”，不建议作为恢复验证样例。

## 四、人工实测步骤

### 步骤 1：确认关闭前状态
记录：
- 当前实例总数
- 当前选中的实例名称
- 设置项当前值

### 步骤 2：完全关闭应用
确保不是只隐藏窗口，而是：
- 真正退出桌面应用进程

### 步骤 3：重新启动应用
重新打开 openclaw-manager。

### 步骤 4：检查恢复结果
逐项核对：

#### 4.1 实例列表
- 关闭前新增的实例仍存在
- 实例数量未丢失
- 实例名称 / 类型 / 地址未被重置

#### 4.2 当前实例
- 应恢复到关闭前最后一次选中的实例
- 不应退回默认本机实例

#### 4.3 设置项
- 关闭前已改动的设置应保持原值
- 不应全部回退为默认值

满足以下全部条件即可判定“重启恢复实测通过”：

## 六、失败判定

出现以下任一情况即判定本轮实测失败：

- 新增实例丢失
- 当前实例恢复错误
- 设置项回退默认值
- 启动时报错或页面异常
- 本地存储 key 存在但 UI 未正确恢复
